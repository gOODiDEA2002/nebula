# Nebula Gateway Core - 发展路线图

> 本文档说明 Nebula Gateway Core 的当前状态、已知问题、计划功能和未来发展方向。

## 当前版本

**版本号**：2.0.1-SNAPSHOT  
**发布时间**：2025-11  
**稳定性**：Beta  
**维护状态**：Active

### 版本特性

- JwtAuthGatewayFilter - JWT认证过滤器
- LoggingGlobalFilter - 全局日志过滤器
- AbstractGrpcServiceRouter - gRPC路由器抽象基类
- GrpcGatewayFilter - HTTP到gRPC桥接
- RateLimitKeyResolver - 多策略限流支持
- GatewayProperties - 完整配置支持

## 已知问题

### 中优先级问题

#### 问题1：gRPC调用超时未可配置

- **问题描述**：gRPC调用使用默认超时，无法通过配置自定义
- **影响范围**：长时间运行的gRPC调用可能超时
- **计划修复版本**：2.0.2

#### 问题2：限流Key解析器互斥

- **问题描述**：当前只能选择一种限流策略，无法组合使用
- **影响范围**：灵活性受限
- **计划修复版本**：2.1.0

### 中期计划（v2.1.x系列）

**发布时间**：2026-Q1  
**重点方向**：功能扩展

#### v2.1.0 (2026-02)

**主要目标**：增强功能

**计划功能**：

##### 功能1：组合限流策略

- **优先级**：高
- **功能描述**：支持同时使用多种限流策略
- **使用场景**：需要IP+用户双重限流

##### 功能2：动态路由

- **优先级**：中
- **功能描述**：支持运行时动态添加/修改路由
- **使用场景**：无需重启即可更新路由配置

##### 功能3：请求/响应转换

- **优先级**：中
- **功能描述**：支持请求/响应格式转换
- **使用场景**：协议适配、数据格式转换

## 优化方向

### 1. 性能优化

#### 1.1 响应速度优化

**目标**：将网关延迟开销控制在5ms以内

**优化措施**：

- [ ] 路由匹配算法优化
- [ ] JWT验证缓存
- [ ] gRPC连接复用

#### 1.2 吞吐量优化

**目标**：支持10万+QPS

**优化措施**：

- [ ] 非阻塞IO优化

- [ ] GC优化

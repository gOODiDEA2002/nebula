### 阶段 0: 修复半透明边框检测 (最高优先级) [当前任务]

#### 问题描述
端到端测试中发现部分六边形拼图验证码无法正确检测：
- 六边形有**白色边框**，内部是**半透明**的（显示背景图案）
- Otsu 阈值方法无法区分六边形和背景
- 导致检测到右侧位置而非左侧目标位置

#### 解决方案
1. [x] 实现白色边框检测算法 (已完成 2026-01-22)
   - 检测高亮度(V>160)、低饱和度(S<80)的像素
   - 只分析图片下半部分（y > 45%）
   - 边缘检测 + 白色区域组合
   - 形态学闭操作连接边框
   - 轮廓分析筛选 4-12 顶点多边形

2. [x] 集成到 `detect_polygon_captcha` (已完成)
   - 作为 Otsu 方法的"备选方法2"
   - 当 Otsu 未检测到形状时自动使用

3. [x] 本地测试验证 (已完成)
   - `/tmp/captcha_bg_original.png` 测试结果:
   - Otsu 方法: 0 个形状 (失败)
   - 白色边框方法: 2 个形状 (成功)
   - 目标位置: 原始 241, 缩放后 121

4. [ ] **待执行: 部署并端到端测试**
